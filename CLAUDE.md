## Architecture Overview

**EDABK_SNN_CIM** is a neuromorphic gesture recognition SoC (ChipFoundry Caravel + BM Labs Neuromorphic X1 ReRAM IP). Three-layer binary-weight LIF SNN classifies 12 DVS128 hand gestures from 256-axon feature vectors. Taped out with 8 ReRAM macros (32 neurons per tile), achieving ~68% hardware accuracy.

See [README.md](README.md) for full project context, [snn_gesture_working.md](snn_gesture_working.md) for system architecture.

### Critical: Two Separate RTL Codebases

| | Simulation Baseline | Tapeout-Ready |
|---|---|---|
| **Path** | `verilog/tb/hdl/` | `verilog/rtl/SNN_gesture/hdl/` |
| **Macros** | 16 (256x64 neurons) | 8 (256x32 neurons, OpenLane-constrained) |
| **Neuron Logic** | Static accumulator | LIF with leak/threshold/reset |
| **Use** | Development, testing | Synthesis, GDS production |

**These are completely independent designs.** Changes to one rarely apply to the other. Always specify which codebase a modification targets.

## Non-Obvious Architectural Decisions

### Parameter Synchronization (Critical)
`NUM_OF_MACRO` is the master tuning knob. It **must stay synchronized** across two systems:
- **RTL truth**: `verilog/rtl/SNN_gesture/hdl/nvm_parameter.vh` (defines `NUM_OF_MACRO=8`, overridden in `user_project_wrapper.v`)
- **Training truth**: `verilog/tb/snn_gesture/nvm_parameter.py` (used by preprocessing, training, weight export, validation)

When either changes, cross-check the other immediately. Mismatch causes connection file dimension errors and HW/SW disagreement.

### Flattened Time SNN (Not Time-Loop Based)
This design encodes all temporal information spatially in the 256-axon input (2 time windows x 11x11 grid, merged polarity). The chip runs one forward pass per gesture and outputs one binary spike per neuron. Despite SNN terminology, it behaves like a static 1-bit ANN.

### Leak Symmetry via Magnitude-Based Rounding (Hardware Invariant)
Original bug: applying leak inside the sequential axon loop created 256x input-order bias (early axons decayed more). Fix in `nvm_neuron_block.v`: compute leak on the absolute value, then sign-correct. This is why `verilog/rtl/SNN_gesture/hdl/nvm_neuron_block.v` uses magnitude-based rounding and why `wbs_we_i_reversed` flip-flop in `nvm_synapse_matrix.v` prevents spurious ACKs.

### 8-Macro Hardware Limitation
Original design targeted 16 macros (64 neurons/tile). OpenLane floor-planning on Caravel's 2920x3520 µm user-project wrapper found only 8 fit without PDN stripe violations. This reduces neurons/tile to 32, requiring more cores per layer (26/8/8 instead of 13/4/4), but the SNN topology remains valid.

See `openlane/SNN_gesture/manual_macro_placement.cfg` for fixed 2x4 macro grid.

## Essential Commands

```bash
# Training Pipeline
cd training

python dvs128_preprocess.py --encoding spatial_focus    # Preprocess DVS128
python train_dvs128.py --epochs 80                       # Train SNN
python export_weights.py --checkpoint checkpoints/best.pt  # Export for hardware
python validate_reference_model.py --checkpoint checkpoints/best.pt  # Verify HW/SW

# RTL Simulation (Tapeout Design)
cd verilog/tb

make sim                                    # Full cocotb test
make sim MODULE=test_neuron_block           # Phase 3 (neuron)
make sim MODULE=test_spike_out              # Phase 4 (spike output)
make sim MODULE=test_single_core            # Phase 5 (end-to-end single core)
make sim MODULE=test_layer_inference        # Phase 6 (layer routing)

# OpenLane Synthesis
cd openlane

make neuron_core                            # Harden macro
make SNN_gesture                            # Harden full wrapper

# Verification
cd verilog/dv
make cocotb                                 # Design verification

# Weights
# Generated by export_weights.py:
# Format: 256 rows x NUM_NEURON cols (MSB-first per row)
# With NUM_OF_MACRO=8: connection_XXX.txt is 256 rows x 32 cols
```

## When Things Go Wrong

- **Connection files wrong dimensions** -> `NUM_OF_MACRO` mismatch between `nvm_parameter.vh` and `nvm_parameter.py`
- **Simulation hangs on picture_done** -> Check `spike_o_padded` width in `nvm_neuron_core_256x64.v` line ~98
- **OpenLane PDN violations** -> 8-macro placement is fixed in `manual_macro_placement.cfg`; do not add more without revisiting floorplan
- **HW/SW accuracy gap** -> See [snn_gesture_working.md](snn_gesture_working.md#accuracy-journey); leak compensation and negative-potential saturation are critical
